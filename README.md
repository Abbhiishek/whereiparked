# ParkSpot

Never lose your car again. A focused, **fully offline** Android app (Expo + React Native) that captures your parking spot in one tap and lets you navigate back with one more.

**Zero cloud. Zero accounts. Zero recurring cost.** Every byte stays on your phone. Uninstall the app and everything is gone.

## Stack

- Expo SDK 54 (`expo-router` v6, New Architecture, React 19)
- `expo-sqlite` + Drizzle ORM for local persistence
- Zustand + MMKV for app preferences
- NativeWind v4 (Tailwind v3 under the hood)
- `react-native-android-widget` for the homescreen widget
- All media (photos, voice notes) stored in app document directory

## Project layout

```
app/                  # expo-router routes
  (onboarding)/       # welcome + permissions priming (first run)
  (tabs)/             # Home / History / Settings
  save.tsx            # capture flow modal
  find.tsx            # navigate back screen
  session/[id].tsx    # history detail
components/
  session/            # capture/playback/timer/CTA
  history/            # list item
  widget/             # Android widget JSX
  ui/                 # primitives (Button, Card)
db/
  schema.ts           # Drizzle schema (one `sessions` table)
  client.ts           # SQLite client (WAL mode)
  migrate.ts          # useMigrations() hook
  migrations/         # generated by drizzle-kit
  queries/            # session query helpers
hooks/                # useLocation, useActiveSession, useRecentSessions
lib/                  # geo, intents, deeplink, retention, storage, time
services/
  notifications/      # local reminder scheduling + action handlers
  widget/             # widget render + update
  exportJson.ts       # share local data as JSON
stores/               # zustand stores (session, settings, onboarding)
types/                # session + widget types
```

## First-run setup

No accounts, no API keys, no Supabase, no Google OAuth. Just install and go.

### Build a dev client

The Android widget requires a custom dev client (not Expo Go).

```
npx eas login
npx eas init
npx eas build --profile development --platform android
```

Install the resulting APK on a device, then run:

```
pnpm start --dev-client
```

Or, if you've run `expo prebuild`, you can build locally:

```
pnpm android
```

## Day-to-day commands

| Command | What it does |
|---|---|
| `pnpm start` | Start Metro |
| `pnpm android` | Build + install on connected Android device |
| `pnpm db:generate` | Regenerate local SQLite migrations from `db/schema.ts` |
| `pnpm db:studio` | Open Drizzle Studio against the local DB |
| `pnpm exec tsc --noEmit` | Type-check the project |
| `pnpm lint` | Run expo lint |

## Architecture notes

- **Local-first, full stop.** SQLite is the only source of truth. There is no remote backup, no sync, no account migration. If you want cross-device, copy the database file manually or use Android's app backup.
- **Notifications**: parking expiration reminders use `Notifications.scheduleNotificationAsync` with a fixed `DATE` trigger. Reliable under Doze.
- **Widget**: the widget render runs in a separate JS context; it reads from the same SQLite file (WAL) as read-only. The widget only deep-links — it never writes.
- **Media**: photos compressed to ~200 KB JPEG (`expo-image-manipulator`), voice notes as `m4a`. Both stored under `${documentDirectory}sessions/${sessionId}/`. Retention sweep purges old files automatically.

## Data export

Settings → Data → Export my data writes a JSON file with every session (coords, times, notes, voice duration) and opens the system share sheet. Useful before uninstalling or moving to a new phone.

## Known gotchas

- pnpm on Windows occasionally trips on its own temp directories during install. Re-running `pnpm install --no-frozen-lockfile` resolves it.
- `react-native-mmkv` v4 uses `createMMKV()` factory and `mmkv.remove(key)` (not `mmkv.delete`).
- `expo-av` is deprecated in SDK 54; the audio recorder still works. Future migration to `expo-audio` is straightforward.
- Drizzle migrations import `.sql` files via `babel-plugin-inline-import`; `metro.config.js` keeps `sql` in `sourceExts`.
